# 2D Roguelike 当前工作状态与后续优化清单

更新时间：2026-05-22

## 一、目前已完成的工作

### 1. 阻断性问题修复

- 修复了进入游戏后玩家无法移动的问题。
- 确认游戏开始时 `Time.timeScale` 会恢复为 `1f`，避免 UI 暂停逻辑遗留导致整局卡死。
- 确认 `PlayerController` 在玩家存活状态下保持启用。
- 检查并清理了会阻挡输入的全屏 UI 遮罩，确保非暂停状态下 UI 不吞掉玩家操作。
- 修复了暂停面板关闭后没有正确恢复游戏时间的问题。

### 2. 玩家 HUD 与 TMP 字体升级

- 将左上角核心 HUD 重构为新的 `ModernHudRoot`。
- 将 HP、XP、GOLD 等主要 HUD 文字升级为 TextMeshPro。
- 导入并生成了可用的 TMP 字体资产：
  - 字体：MedievalSharp
  - 路径：`Assets/Resources/Fonts/MedievalSharp/MedievalSharp SDF.asset`
  - 许可：OFL，可免费商用
- 修复了之前 TMP 字体资产缺少 Atlas / Material 导致 HUD 只显示部分文本的问题。
- GOLD 数字已加大，并加入描边、投影和缩放反馈。
- 金币变化已改为滚动数字效果，而不是瞬间跳变。

### 3. 血条与状态反馈

- 实现了玩家 HP 的缓冲血条逻辑。
- 玩家受伤后，主血条立即扣减，缓冲条延迟后平滑追随。
- 左上角 HUD 已隐藏旧的玩家世界血条，减少画面杂乱。

### 4. UI 动效与层级管理

- 新增或接入了 `UIAnimationUtility`，用于 UI 的淡入、弹出、缩放反馈。
- ESC 暂停面板已支持平滑弹出和淡入淡出。
- 打开暂停面板时游戏暂停，关闭后恢复。
- 新增全局遮罩 `UIModalMask`，用于弹窗打开时统一控制背景暗化。

### 5. 场景交互提示

- 木桶等可交互物体接入了 `InteractableHint`。
- 玩家靠近木桶时，头顶会出现 World Space 提示。
- 木桶提示文字已改为 `Press E Break`。
- 木桶交互输入支持：
  - `E`
  - `J`
  - 鼠标左键

### 6. 战斗与物件体验修复

- 修复过敌人攻击朝向不正确的问题，使攻击前摇和攻击瞬间更稳定地面向玩家。
- 修复过击杀后进入慢动作但不恢复的问题。
- 修复过场景中木桶、箱子、柱子等可见物体碰撞体丢失的问题。
- 优化过敌人受击、死亡、玩家受伤等反馈逻辑，包括屏幕闪红、闪白、受击粒子、击退等方向。

- `Time.timeScale = 1`。

## 二、当前仍需要优化的地方

### 1. UI 美术质感仍需继续打磨

当前 HUD 已经从默认程序员 UI 升级为 TMP + 统一面板风格，但还没有达到成熟商业独立游戏的完成度。

建议继续优化：

- 为 HUD 使用真正的九宫格像素边框或中世纪 UI 框体。
- 给 HP、XP、GOLD 增加小图标，而不是只靠文字区分。
- GOLD 区域可以加入金币 Sprite，并在拾取金币时做小幅闪光或跳动。
- 暂停面板、升级面板、遗物面板需要统一视觉语言。
- 旧 UI 面板需要彻底替换为新版 TMP + 统一皮肤，而不是只隐藏或局部升级。

### 2. 字体本地化支持不足

当前使用的 MedievalSharp 适合英文地牢风格，但不支持中文。

如果后续 UI 需要中文显示，建议：

- 额外导入一款支持中文的可商用字体。
- 为中文和英文分别建立 TMP Font Asset。
- 配置 TMP fallback font chain，避免中文缺字。
- 所有运行时提示文本统一走本地化表，而不是写死在脚本里。

### 3. 交互提示需要更成熟的交互系统

目前 `InteractableHint` 可以显示提示，但系统还偏基础。

建议继续优化：

- 增加统一的 `IInteractable` 接口。
- 由玩家交互检测器统一选择最近的可交互物体。
- 同屏多个交互物时，只显示当前最近或最高优先级目标的提示。
- 提示文字跟随输入设备变化，例如键盘显示 `E`，手柄显示对应按钮图标。
- 加入按键图标 Sprite，而不是纯文本提示。

### 4. 战斗打击感还可以继续增强

当前已经加入受击反馈方向，但整体仍有提升空间。

建议继续优化：

- 区分普通攻击、暴击、死亡、破盾等不同粒子。
- 敌人死亡时使用更明确的溶解、碎裂或尸体残留表现。
- 玩家受击时加入短暂 hit stop，但必须保证有自动恢复保护。
- 攻击命中时加入轻微摄像机震动。
- 伤害数字根据伤害类型使用不同颜色和弹出节奏。

### 5. 对象池与资源管理需要统一

目前部分特效、伤害数字、交互提示已经考虑对象池方向，但仍需要统一管理。

建议继续优化：

- 建立统一的 `PoolManager`。
- 粒子、伤害数字、金币、飘字、临时 UI 全部走对象池。
- 对池对象设置自动回收时间。
- 避免战斗高峰时频繁 Instantiate / Destroy。

### 6. QA 自动化仍需加强

当前 QA 主要通过 Unity MCP 和手动 Play Mode 脚本验证。

建议继续补充：

- 增加一个开发用 QA 场景，包含玩家、木桶、箱子、敌人、金币、升级触发器。
- 增加 Debug 面板，可一键测试：
  - 加金币
  - 扣血
  - 升级
  - 生成敌人
  - 触发暂停
  - 触发死亡
- 增加启动时自检脚本，检查：
  - `Time.timeScale`
  - `EventSystem`
  - PlayerController 是否启用
  - 关键 UI 引用是否为空
  - Console 是否出现致命错误

## 三、建议下一阶段优先级

### P0：保证游戏流程稳定

- 每次修改 UI 或暂停逻辑后，必须验证进入游戏、移动、攻击、暂停、恢复。
- 所有会修改 `Time.timeScale` 的代码必须有恢复路径。
- 所有弹窗关闭时必须恢复输入和游戏时间。

### P1：彻底统一 UI 风格

- 替换升级面板、遗物面板、死亡面板、暂停面板的旧 Text 和旧 Image。
- 建立统一的 UI Prefab 或 UI 工厂方法。
- 为按钮、面板、标题、数值、提示统一字号、颜色和描边规则。

### P2：强化战斗反馈

- 玩家受击：闪白、屏幕红闪、击退、血条缓冲联动。
- 敌人受击：粒子、击退、伤害数字、短暂停顿。
- 敌人死亡：专属死亡特效、尸体淡出、掉落物弹出。

### P3：优化交互提示

- 加入按键图标。
- 支持键鼠和手柄输入切换。
- 多交互目标时只显示一个最合理的提示。
- 离开交互范围后提示必须稳定消失。
